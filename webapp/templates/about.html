Project Overview

AI-Based Loan Default Prediction System

This system predicts whether a loan applicant is likely to default based on their financial profile. A trained XGBoost classifier processes 36 engineered features from real lending data, wrapped in a Flask web application with SHAP-based explainability and PSI-based data drift monitoring.

Tech Stack
Python 3.x Flask XGBoost scikit-learn SHAP pandas NumPy imbalanced-learn joblib matplotlib
ML Pipeline — 8 Stages
01

Data Ingestion

Raw CSV loaded from data/raw/loan_dataset.csv. Shape, dtype, and null counts validated on startup.

02

Data Cleaning

Deduplication, median imputation for numeric columns, mode imputation for categorical columns.

03

Feature Engineering

Derives debt-to-income ratio, loan-to-income ratio, credit utilization, employment stability, and income category.

04

Encoding & Scaling

LabelEncoder for categoricals. StandardScaler fitted on training set, applied to test set. Scaler serialized via joblib.

05

Class Balancing

SMOTE (Synthetic Minority Over-sampling Technique) applied to training set to handle the natural class imbalance in lending data.

06

Model Training

Three models compared — Logistic Regression, Random Forest, XGBoost — with 5-fold cross-validation. Best model saved via joblib.

07

SHAP Explainability

SHAP TreeExplainer generates per-prediction feature attribution. Summary and force plots exported to outputs/.

08

Drift Monitoring

PSI computed for 5 key features between training distribution and new data. Alerts fired when PSI > 0.25.

How to Run

Step-by-step execution order

# 1. Install dependencies
pip install -r requirements.txt

# 2. Preprocess data (clean, engineer, encode, SMOTE)
python src/data_preprocessing.py

# 3. Train and save best model
python src/train_model.py

# 4. Evaluate model (generates plots in outputs/)
python src/evaluate_model.py

# 5. Generate SHAP + fairness reports
python src/shap_explainer.py

# 6. Run drift monitoring
python monitoring/drift_detection.py

# 7. Start the web app
python webapp/app.py
# → Open: http://localhost:5000

Fairness & Ethics

Demographic parity and equal opportunity metrics

The shap_explainer.py module computes two fairness metrics across the home_ownership sensitive attribute to detect discriminatory model behaviour.

Demographic Parity
Measures whether the positive prediction rate is equal across groups. A large gap indicates potential selection bias against certain groups.
Equal Opportunity
Compares True Positive Rate (recall) across groups. Unequal TPR means the model makes different error types depending on group membership.
