Project Overview AI-Based Loan Default Prediction System

This system predicts whether a loan applicant is likely to default based on their financial profile. A trained XGBoost classifier processes engineered features from real lending data, wrapped in a Flask web application with SHAP-based explainability and PSI-based drift monitoring.

How to Run Step-by-step execution order 
# 1. Install dependencies
pip install -r requirements.txt

# 2. Preprocess data (clean + engineer features)
python -m src.data_preprocessing

# 3. Train XGBoost model (includes SMOTE + GridSearchCV)
python -m src.train_model

# 4. Evaluate model (optional — overwrites model_metrics.json)
python -m src.evaluate_model

# 5. Generate SHAP + fairness reports
python -m src.shap_explainer

# 6. Run drift monitoring
python -m monitoring.drift_detection

# 7. Start the web app
python -m webapp.app
# → Open: http://127.0.0.1:5000

# Docker (alternative)
docker compose up --build

Fairness & Ethics Demographic parity and equal opportunity metrics
Demographic Parity
Measures whether the positive prediction rate is equal across groups. A large gap indicates potential selection bias.
Equal Opportunity
Compares True Positive Rate per group. Unequal TPR means the model makes different error types by group membership.
