# Credit Risk Prediction System

An end-to-end machine learning pipeline that predicts the probability a loan applicant will default, using a single **XGBoost** model with SHAP-based explainability and a Flask web UI.

---

## Project Structure

```
credit-risk-prediction/
├── data/
│   ├── raw/                  ← Put loan_dataset.csv here
│   └── processed/            ← Auto-generated by data_preprocessing.py
├── models/                   ← Saved model artefacts
├── outputs/                  ← SHAP plots, drift charts, reports
├── reports/                  ← Per-borrower risk reports (.txt)
```

## Quick Start

### 1 — Install dependencies

```bash
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 2 — Add your dataset

Place the LendingClub-format CSV in:
```
data/raw/loan_dataset.csv
```

### 3 — Run the pipeline

```bash
# Step 1: preprocess
python -m src.data_preprocessing

# Step 2: train (XGBoost, GridSearchCV, SMOTE)
python -m src.train_model

# Step 3: evaluate (optional — overwrites model_metrics.json)
python -m src.evaluate_model

# Step 4: generate SHAP + fairness outputs
python -m src.shap_explainer

# Step 5: launch Flask app
python -m webapp.app
```

Open http://127.0.0.1:5000 in your browser.

## Key Design Decisions

| Decision | Rationale |
|---|---|
| Single model (XGBoost) | Removes complexity of multi-model selection; XGBoost is best-suited for tabular credit data |
| SMOTE applied on training split only | Prevents data leakage into the test set |
| Optimal threshold via Youden's J | Maximises recall without completely sacrificing precision — critical for default detection |
| SHAP TreeExplainer | Exact Shapley values for tree models; consistent between training and inference |
| PSI-based drift detection | Industry-standard method; actionable threshold (≥ 0.25 → retrain) |
| Append-only audit log | Governance requirement; every decision persists with its raw inputs |

## Dataset

The pipeline expects a LendingClub-format CSV with these minimum columns:

`loan_amnt`, `int_rate`, `installment`, `annual_inc`, `dti`, `fico_range_low`, `fico_range_high`, `open_acc`, `revol_bal`, `total_acc`, `revol_util`, `delinq_2yrs`, `inq_last_6mths`, `pub_rec`, `term`, `grade`, `sub_grade`, `emp_length`, `home_ownership`, `verification_status`, `purpose`, `addr_state`, `loan_status`

`loan_status` must be binary: `0 = Repay`, `1 = Default`.
